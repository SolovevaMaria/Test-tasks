                                                        Инструкция пользователя.
                                            Система управления производственными заказами.

1). Введение.

Система управления производственными заказами предназначена для автоматизации процессов создания, редактирования и отслеживания производственных заказов. Система предоставляет REST API для интеграции с другими приложениями.

          Основные возможности:
- Управление справочником марок стали
- Создание и редактирование производственных заказов
- Управление позициями заказов
- Поиск заказов по различным критериям
- Отслеживание статусов заказов и позиций

2). Начало работы

- Базовый URL API:
http://<сервер>:8080/api/

Для локального тестирования: `http://localhost:8080/api/`

- Статусы заказа:
- 'новый' - заказ создан, работы не начаты
- 'в работе' - производство начато
- 'выполнен' - заказ полностью выполнен

- Статусы позиций:
- 'новая' - позиция создана
- 'в работе' - производство позиции начато
- 'выполнена' - позиция произведена

3). Работа с марками стали. Запросы:

Получить все марки стали:
GET /api/steel-grades

Получить марку стали по ID:
GET /api/steel-grades/{id}

Создать новую марку стали:
POST /api/steel-grades

Обновить марку стали:
PUT /api/steel-grades/{id}

Удалить марку стали:
DELETE /api/steel-grades/{id}

4). Работа с заказами. Запросы:

Получить все заказы:
GET /api/orders
 
Получить заказ по ID:
GET /api/orders/{id}

Создать новый заказ:
POST /api/orders

Обновить заказ:
PUT /api/orders/{id}

Удалить заказ:
DELETE /api/orders/{id}

5). Поиск заказов. Запросы:

Поиск по цеху-производителю
POST /api/orders/search

Поиск по марке стали:
{
    "steelGrade": "Ст3сп"
}

Поиск по диаметру:
{
    "diameter": 100.50
}

Комбинированный поиск:

{
    "workshop": "Цех",
    "steelGrade": "Ст3сп",
    "diameter": 100.50,
    "wallThickness": 5.25
}

6). Коды ответов API.

| Код  | Описание                                |
|------|-----------------------------------------|
| 200  | OK - успешный запрос                    |
|------|-----------------------------------------|
| 201  | -Created - ресурс создан                |
|------|-----------------------------------------|
| 204  | No Content - успешное удаление          |
|------|-----------------------------------------|
| 400  | -Bad Request - ошибка в данных          |
|------|-----------------------------------------|
| 404  | Not Found - ресурс не найден            |
|------|-----------------------------------------|
| 500  | -Internal Server Error - ошибка сервера |
|------|-----------------------------------------|
