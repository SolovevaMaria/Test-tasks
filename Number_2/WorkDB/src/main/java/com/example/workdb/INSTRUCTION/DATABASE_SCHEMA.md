                                                Физическая модель данных.
                                      Система управления производственными заказами.

1). Общая структура базы данных.
 Название базы данных: `ProductionOrdersDB`

2). Таблицы.

- Таблица SteelGrades (Марки стали)
|------|-----|----------------------------|--------------------------|
| Поле | Тип | Ограничения                | Описание                 |
|------|-----|----------------------------|--------------------------|
| Id   | INT | IDENTITY(1,1), PRIMARY KEY | Уникальный идентификатор |
|------|-----|----------------------------|--------------------------|
| Name | NVARCHAR(100) | NOT NULL, UNIQUE | Наименование марки стали |
|------|-----|----------------------------|--------------------------|

- Таблица Orders (Заказы).
|-------------|---------------|----------------------------|----------------------|
| Поле        | Тип           | Ограничения                | Описание             |
|-------------|---------------|----------------------------|----------------------|
| Id          | INT           | IDENTITY(1,1), PRIMARY KEY | Идентификатор заказа |
|-------------|---------------|----------------------------|----------------------|
| OrderNumber | NVARCHAR(50)  | NOT NULL, UNIQUE           | Номер заказа         |
|-------------|---------------|----------------------------|----------------------|
| Workshop    | NVARCHAR(100) | NOT NULL                   | Цех-производитель    |
|-------------|---------------|----------------------------|----------------------|
| StartDate   | DATE          | NOT NULL                   | Дата начала          |
|-------------|---------------|----------------------------|----------------------|
| EndDate     | DATE          | NOT NULL                   | Дата окончания       |
|-------------|---------------|----------------------------|----------------------|
| Status      | NVARCHAR(20)  | NOT NULL, DEFAULT 'новый'  | Статус заказа        |
|-------------|---------------|----------------------------|----------------------|

- Таблица OrderItems (Позиции заказа).
|---------------|---------------|----------------------------|-----------------------|
| Поле          | Тип           | Ограничения                | Описание              |
|---------------|---------------|----------------------------|-----------------------|
| Id            | INT           | IDENTITY(1,1), PRIMARY KEY | Идентификатор позиции |
|---------------|---------------|----------------------------|-----------------------|
| OrderId       | INT           | NOT NULL, FOREIGN KEY      | Ссылка на заказ       |
|---------------|---------------|----------------------------|-----------------------|
| ItemNumber    | INT           | NOT NULL                   | Номер позиции         | 
|---------------|---------------|----------------------------|-----------------------|
| SteelGradeId  | INT           | NOT NULL, FOREIGN KEY      | Ссылка на марку стали |
|---------------|---------------|----------------------------|-----------------------|
| Diameter      | DECIMAL(10,2) | NOT NULL                   | Диаметр               |
|---------------|---------------|----------------------------|-----------------------|
| WallThickness | DECIMAL(10,2) | NOT NULL                   | Толщина стенки        |
|---------------|---------------|----------------------------|-----------------------|
| Quantity      | DECIMAL(15,2) | NOT NULL                   | Объем                 |
|---------------|---------------|----------------------------|-----------------------|
| Unit          | NVARCHAR(20)  | NOT NULL                   | Единица измерения     |
|---------------|---------------|----------------------------|-----------------------|
| Status        | NVARCHAR(20)  | NOT NULL, DEFAULT 'новая'  | Статус позиции        |
|---------------|---------------|----------------------------|-----------------------|

3). Индексы.

Поиск по цеху:
CREATE INDEX IX_Orders_Workshop ON Orders(Workshop);

Фильтрация по статусу:
CREATE INDEX IX_Orders_Status ON Orders(Status);

Связь с марками стали:
CREATE INDEX IX_OrderItems_SteelGrade ON OrderItems(SteelGradeId);

Поиск по характеристикам:
CREATE INDEX IX_OrderItems_Diameter ON OrderItems(Diameter);
CREATE INDEX IX_OrderItems_WallThickness ON OrderItems(WallThickness);


4). Связи между таблицами.

┌─────────────┐         ┌─────────────┐
│   Orders    │         │ OrderItems  │
├─────────────┤         ├─────────────┤
│ Id (PK)     │────┐    │ Id (PK)     │
│ OrderNumber │    │    │ OrderId (FK)│────┐
│ Workshop    │    └────┤ ItemNumber  │    │
│ StartDate   │         │ SteelGradeId│────┐
│ EndDate     │         │ ...         │    │
│ Status      │         └─────────────┘    │
└─────────────┘                            │
                                           │
                                ┌──────────┴──────────┐
                                │                     │
                                ▼                     ▼
                        ┌─────────────┐      ┌─────────────┐
                        │ SteelGrades │      │   Индексы   │
                        ├─────────────┤      │   поиска    │
                        │ Id (PK)     │      └─────────────┘
                        │ Name        │
                        └─────────────┘


Типы связей:
- Orders → OrderItems: Один ко многим (1 ко многим)
- SteelGrades → OrderItems: Один ко многим (1 ко многим)
